# راهنمای تست و TDD فاز دوم

نوید جان، برای اینکه فاز دوم فقط ماژولار و متادیتا‌محور نباشد و از نظر تست هم قابل دفاع باشد، یک لایه تست سبک به پروژه اضافه شده است.

هدف این تست‌ها این نیست که کل پروژه را سنگین کنیم؛ هدف این است که بخش‌های حساس فاز دوم قبل از Merge نهایی قابل کنترل باشند:

- تشخیص سؤال HR و غیر HR
- جلوگیری از خروجی فردی و اطلاعات حساس
- تشخیص Data Gap مثل تحلیل شهری
- نگاشت مفاهیم فارسی به ستون‌های View
- انتخاب Route درست بین SQL / Gap / Reject
- تولید SQL از Template
- اعتبارسنجی SQL قبل از اجرا
- تست End-to-End سبک روی Orchestrator بدون نیاز به دیتابیس واقعی

## مسیر تست‌ها

```text
backend/tests/
```

فایل‌های اصلی:

```text
test_metadata_service.py
test_domain_and_validator.py
test_semantic_router_templates.py
test_sql_validator.py
test_orchestrator.py
```

## اجرای تست‌ها

از مسیر `backend` اجرا شود:

```bash
pip install -r requirements.txt
pytest tests -q
```

## نکته مهم

این نسخه را می‌شود `TDD-ready` دانست، نه اینکه کل پروژه از روز اول با TDD کلاسیک نوشته شده باشد.
یعنی برای ماژول‌های حساس تست اجرایی اضافه شده و از این به بعد بهتر است هر تغییر روی Metadata، SQL Template، Router یا Validator با همین تست‌ها کنترل شود.

## معیار تصمیم

اگر این تست‌ها Pass شوند، Merge فاز دوم به پروژه اصلی کم‌ریسک‌تر است.
اگر تست‌ها Fail شوند، اول همان مورد باید اصلاح شود و بعد Merge انجام شود.
